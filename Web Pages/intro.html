Introduction


html stand for Hypertext Markup Language, a language that is used for creating web pages.html is not a programming language. It is a layout specification language in which elements called Tags and their Attributes are used to markup (or specify) how the various types of contents such as text, pictures, tables, audio, video, etc., will be displayed by a web browser.

The html documents have two basic components: contents (text, images, music and video clips,etc.) and tags. html documents can be created in any simple text editor like Notepad and must have a .htm or .html file extension. Web Browser like the Internet Explorer or FireFox to read the html documents and display them as web pages.



html Tags and Attributes

 A Tag consists of a keyboard (and many times also the attributes) enclosed within a left angled bracket (<) and a right angled bracket (>). It is a formatting specification which tells a web browser how to display a particular content (text, picture, video, audio, etc.) it marks up.
The majority of tags have attributes. The attributesare the sub elements or features of the tags that give additional functionality to the tags, allowing a user to manipulate the marked up elements in different ways.




Types of Tags

 The html tags can be classified into two main categories - Container Tags and Empty Tags.

Container Tags: Container Tags are used in pairs and have a starting tag (ON tag) and ending tag (OFF tag). These tags define how the contents contained within them should be formatted.
Example:To make a line of the text bold, we use the tags <b>...</b>;
where <b> is the starting tag and </b> is the ending tag and the ellipse (...) represent the text to be made bold.

Empty Tags have onlly a starting tag. They do not need to be followed by an ending tag. Empty tags are used for giving formatting constructs.
Eample:To insert a line break, we use the tag <br>. Just writing <br> after the required text tells the browser to start any text that follows from the next line.


REMEMBER : Tags are not case sensitive. So <b>...</b> is the same as <B>...</B>, but the correct way is to type them in lower case.
